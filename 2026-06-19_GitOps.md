## 3 Practice Interview Questions

**1. Technical**
> "Walk me through how ArgoCD syncs a Helm chart change from a PR merge to production."

**2. Scenario-based**
> "Your ArgoCD app shows 'OutOfSync' but no PR has been merged recently. How do you investigate and resolve it?"

**3. Behavioral**
> "Tell me about a time a GitOps deployment caused an incident. What happened, and what did you change afterwards?"

---

### 💡 Suggested Answer Outline — Q2 (OutOfSync scenario)

1. **Check ArgoCD UI / CLI first** — `argocd app get <app> --hard-refresh` to rule out stale cache
2. **Identify the diff** — `argocd app diff <app>` to see exactly what's drifted
3. **Trace the cause** — Was it a manual `kubectl apply`? A mutating webhook? An operator patching a field ArgoCD doesn't manage?
4. **Short-term fix** — If safe, sync back: `argocd app sync <app>`; if risky, add the field to `ignoreDifferences` in the Application spec
5. **Root cause remediation** — Enforce RBAC to block direct kubectl apply in prod; document the incident; add a Slack alert for OutOfSync > 5 min
6. **Process change** — Add a reconciliation check in CI to catch drift in staging before it hits prod

## One Hands-on Task (~45 min)

**Build a local GitOps loop with ArgoCD + kind**

```bash
# 1. Create a local cluster
kind create cluster --name gitops-demo

# 2. Install ArgoCD
kubectl create namespace argocd
kubectl apply -n argocd -f https://raw.githubusercontent.com/argoproj/argo-cd/stable/manifests/install.yaml

# 3. Wait for pods
kubectl wait --for=condition=available deployment/argocd-server -n argocd --timeout=120s

# 4. Port-forward the UI
kubectl port-forward svc/argocd-server -n argocd 8080:443 &

# 5. Get admin password
kubectl get secret argocd-initial-admin-secret -n argocd -o jsonpath="{.data.password}" | base64 -d

# 6. Create an Application pointing to a public Helm chart repo
#    (e.g. bitnami/nginx) and watch ArgoCD sync it

# 7. DRIFT EXERCISE: manually patch the deployment replica count
kubectl scale deployment nginx --replicas=3 -n default
# Observe ArgoCD detect drift → sync back → replicas return to 1
```

**Goal:** See the full sync loop, trigger manual drift, and watch ArgoCD self-heal. Then add a `SyncPolicy: automated` to your Application YAML and repeat.
